# Project Instructions for Gemini

This file contains custom instructions, guidelines, and preferences for Gemini when assisting with the Hotel Management project.

## Project Overview
- **Project Name:** Hotel Management (Lumière Palace)
- **Architecture:** Monorepo with Java Spring Boot backend (`code/backend`) and Next.js frontend (`code/frontend`).

## Core Tech Stack
- **Backend:** Java, Spring Boot (Maven)
- **Frontend:** Next.js 15 (React 19), TypeScript, Tailwind CSS v4, Axios for API calls.

---

## UI & Styling Guidelines (Lumière Palace Theme)

To ensure visual consistency and a premium, luxury aesthetic across the application, all UI development must adhere to the design system shown in the mockup:

### 1. Color Palette
- **Sidebar & Dark Backgrounds:** `#1C1613` (Rich Dark Espresso Brown).
- **Main App Background:** `#FAF9F6` (Warm Alabaster / Cream).
- **Primary / Brand Accent Gold:** `#C5A86E` (Muted Gold) - used for active borders, trend lines, specific numbers, and highlights.
- **Card Backgrounds:** `#FFFFFF` (Pure White).
- **Text Color:** `#1C1613` (Primary Text), `#766E65` (Secondary/Muted Text).
- **Badges:**
  - Positive change (e.g. `+12.4%`): Green (`bg-emerald-50 text-emerald-600 border border-emerald-100`)
  - Negative change (e.g. `-2.1%`): Red (`bg-rose-50 text-rose-600 border border-rose-100`)

### 2. Typography
- **Headings & Main Numbers:** Use an elegant Serif font (e.g., Google Fonts `Playfair Display` or `Cormorant Garamond`) to establish a luxury identity.
- **Body & Secondary Labels:** Use a clean, modern Sans-serif font (e.g., Google Fonts `Plus Jakarta Sans` or `Inter`) for maximum readability.

### 3. UI Component Specifications
- **Left Navigation Sidebar:**
  - Dark background (`#1C1613`).
  - Active item: Background highlighting or subtle gold left border.
  - Badges (like notifications count): Gold circles with white text.
- **Metrics/KPI Cards:**
  - Pure white background with rounded corners (`rounded-2xl` or `20px` radius).
  - Very soft, warm-toned shadow.
  - KPI values should be large and styled with the elegant Serif font.
  - Includes a top-right trending badge (positive green or negative red) with trend arrows (`↗` or `↘`).
- **Charts & Visualizations:**
  - Custom colors for Pie/Donut charts: Blue, Gold, Green, Orange/Red.
  - Line charts: Smooth curve (monotone) in Primary Accent Gold (`#C5A86E`) with a soft matching gradient fill.

## Coding Standards & Preferences

### 1. Frontend Rules (Next.js & React 19)
- **App Router:** Follow Next.js App Router conventions. Place page files in `src/app/**/page.tsx` and custom layouts in `layout.tsx`.
- **TypeScript:** Use TypeScript strictly. Never use `any`. Always declare proper interfaces/types for props and API responses.
- **Component Styling:** Use Tailwind CSS utility classes. Avoid inline styles or custom CSS unless Tailwind cannot achieve the desired result.
- **State Management:** Keep state local to the component unless shared state is required. Use custom hooks for complex state logic.
- **API Fetching:** Use the Axios instance configured in the project. Wrap all API requests in service functions (e.g., in a `src/services` folder) rather than calling Axios directly inside components.
- **React Components:** Use functional components and arrow function syntax:
  ```tsx
  import React from 'react';

  interface ButtonProps {
    label: string;
    onClick: () => void;
  }

  export const Button = ({ label, onClick }: ButtonProps) => {
    return (
      <button 
        onClick={onClick}
        className="px-4 py-2 bg-blue-600 text-white rounded-md hover:bg-blue-700 transition"
      >
        {label}
      </button>
    );
  };
  ```

### 2. Backend Rules (Spring Boot)
- **Architecture:** Follow the Controller-Service-Repository pattern.
- **REST APIs:** Use standard HTTP methods (`GET`, `POST`, `PUT`, `DELETE`). Return JSON payloads with proper status codes (e.g., `200 OK`, `201 Created`, `400 Bad Request`, `404 Not Found`).
- **Dependency Injection:** Prefer constructor injection over field injection (`@Autowired` on fields).
- **Naming Conventions:** Standard Java conventions (CamelCase for classes, camelCase for methods/variables, UPPERCASE for constants).
- **Database/JPA:** Ensure entity mapping is clean and avoid lazy loading issues (e.g., using DTOs instead of exposing raw entities directly in controllers).

### 3. General & AI Interaction Preferences
- **Language:** Respond in Vietnamese.
- **Explanations:** Explain the approach briefly before writing or changing code.
- **Refactoring:** When modifying code, keep unrelated code structures and comments intact.
- **Security:** Do not commit hardcoded credentials, API keys, or JWT secrets. Use `.env` or system variables.
